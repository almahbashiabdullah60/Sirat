# نظام تصفية المحتوى المتقدم - نظرة عامة ومعمارية النظام

يقدم هذا المستند نظرة عامة شاملة على **نظام تصفية المحتوى المتقدم** الذي سيتم دمجه في تطبيق **صراط** (Sirat). تم تقسيم الخطة إلى 5 مراحل نمطية (Modular)، مما يتيح لأي محرر أكواد ذكاء اصطناعي البدء في أي مرحلة وتطبيقها بشكل مستقل دون الخروج عن السياق.

---

## 🏗️ التصميم المعماري للنظام

يعمل النظام كمحرك تصفية DNS محلي (Local DNS Proxy) على جهاز المستخدم. من خلال استخدام خدمة `VpnService` الخاصة بنظام أندرويد، يتم اعتراض طلبات الـ DNS (المنفذ 53 لبروتوكول UDP) وتصفيتها محلياً. يتم تحليل حزم DNS باستخدام مكتبة **dnsjava** بدلاً من المعالجة اليدوية للبايتات، مما يضمن الاستقرار والسرعة.

يتم تضمين قوائم النطاقات المحجوبة داخل التطبيق كملفات JSON في مجلد `assets/filter/` (الملفات موجودة في `plan/` وتُنقل إلى `assets/filter/` عند التنفيذ). كل مدخل JSON بصيغة `"0.0.0.0 domain.com"` — يتم تنظيف البادئة تلقائياً في `SyncManager`. مع إمكانية التحديث عبر URL قابل للتكوين (اختياري). تُحفظ النطاقات محلياً في قاعدة بيانات **Room** للفحص الفوري.

**هام - التفاعل مع الخدمات الحالية:** خدمة الـ VPN هذه مستقلة تماماً عن خدمات قفل التطبيقات (`AppLockAccessibilityService`، `ShizukuAppLockService`، `UsageLockService`). لا تتداخل معها ولا تعتمد عليها. يمكن للمستخدم تفعيل التصفية VPN بشكل منفصل عن قفل التطبيقات. يتم تنسيق الحالة عبر `AppLockManager` فقط لمنع تعارض الخدمات (لا يمكن تشغيل VPN + Shizuku معاً إذا تطلب Shizoku وصولاً للشبكة).

```text
+-----------------------------------------------------------------------------------+
|                                 جهاز الأندرويد                                    |
|                                                                                   |
|  +--------------------+                         +-------------------------+       |
|  |     التطبيقات       |                         |  قاعدة البيانات المحلية  |       |
|  | (المتصفح، يوتيوب إلخ)|                         |      (Room Database)    |       |
|  +---------+----------+                         +------------+------------+       |
|            |                                                 ^                    |
|            | (طلب DNS: منفذ 53)                              |                    |
|            v                                                 | (تحميل من assets   |
|  +---------+----------+      فحص النطاق       +------------+ |  أو HTTP عند بدء   |
|  |     VpnService     |---------------------> |   Filter   | |  التشغيل)          |
|  |   (dnsjava parser) |<--------------------- | Repository | |                    |
|  +--------------------+  (حظر أو تمرير عبر    +------------+ |                    |
|                           UDP socket حقيقي)                  |                    |
|                                                              |                    |
+--------------------------------------------------------------v--------------------+
```

### المكونات الأساسية للنظام:
1. **طبقة الشبكة والـ VPN (`services/vpn/`):**
   - `SiratVpnService.kt`: تدير دورة حياة الـ VPN المحلي، وتنشئ واجهة الـ TUN الافتراضية.
   - `VpnController.kt`: إدارة الصلاحيات والتشغيل/الإيقاف من واجهة المستخدم.
2. **طبقة تحليل الـ DNS (`services/vpn/`):**
   - `DnsResolver.kt`: تستخدم مكتبة **dnsjava** لتحليل طلبات DNS الواردة، واستخراج النطاق، وبناء الرد (NXDOMAIN للمحظور، أو تمرير الطلب لخادم DNS عام).
3. **طبقة البيانات المحلية (`data/filter/`):**
   - `FilterDatabase.kt` & `FilterDao.kt`: قاعدة بيانات Room محلية لتخزين النطاقات النشطة.
   - `FilterRepository.kt`: مستودع للربط بين قاعدة البيانات وذاكرة الكاش (HashSet) السريعة.
   - `SyncManager.kt`: تحميل قوائم النطاقات من ملفات `assets/` أو من URL عند بدء الخدمة.
4. **طبقة فرض البحث الآمن (`services/vpn/`):**
   - إعادة توجيه استعلامات محركات البحث إلى نطاقات البحث الآمن عبر DNS.
5. **واجهة المستخدم (`features/filtering/ui/`):**
   - واجهة Jetpack Compose مع Material 3 (باستخدام Theme التطبيق الحالي) لإدارة إعدادات التصفية وتفعيل الخدمة وعرض السجلات.

## ⚠️ قوانين هامة لمطوري ومحرري الأكواد:
* **لغة البرمجة:** Kotlin بالكامل في جانب الأندرويد.
* **الأداء الخفيف:** يجب ألا تتسبب تصفية طلبات DNS في أي تأخير ملحوظ في سرعة اتصال المستخدم بالإنترنت.
* **الخصوصية أولاً:** لا يتم تسجيل أو إرسال أي نطاقات يزورها المستخدم إلى الخادم، فقط يتم تحميل قوائم الحجب من ملفات assets المضمنة أو URL مخصص عند بدء التشغيل.
* **عدم التعارض مع الخدمات الحالية:** خدمة VPN مستقلة تماماً عن `AppLockAccessibilityService` و `ShizukuAppLockService` و `UsageLockService`. لا تؤثر على آلية كشف التطبيقات المفتوحة ولا تتأثر بها.
