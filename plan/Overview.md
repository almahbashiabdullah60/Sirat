# نظام تصفية المحتوى المتقدم - نظرة عامة ومعمارية النظام

يقدم هذا المستند نظرة عامة شاملة على **نظام تصفية المحتوى المتقدم** الذي سيتم دمجه في تطبيق **صراط** (Sirat). تم تقسيم الخطة إلى 6 مراحل نمطية (Modular) باللغة العربية، مما يتيح لأي محرر أكواد ذكاء اصطناعي البدء في أي مرحلة وتطبيقها بشكل مستقل دون الخروج عن السياق.

---

## 🏗️ التصميم المعماري للنظام

يعمل النظام كمحرك تصفية DNS محلي (Local DNS Proxy) على جهاز المستخدم. من خلال استخدام خدمة `VpnService` الخاصة بنظام أندرويد، يتم اعتراض جميع طلبات الـ DNS (المنفذ 53 لبروتوكول UDP) وتصفيتها محلياً دون إرسال بيانات التصفح إلى خوادم خارجية. 

إضافة إلى ذلك، يتم إعداد مستودع ملفات ساكنة على **GitHub Pages** يحتوي على ملفات JSON مضغوطة للنطاقات المحجوبة (`porn.json`, `gambling.json`, `social.json`) مع ملف ميتاداتا لتتبع الإصدار، ويقوم تطبيق أندرويد بمزامنة هذه الملفات وحفظها محلياً في قاعدة بيانات **Room**.

```text
+-----------------------------------------------------------------------------------+
|                                 جهاز الأندرويد                                    |
|                                                                                   |
|  +--------------------+                         +-------------------------+       |
|  |     التطبيقات       |                         |  قاعدة البيانات المحلية  |       |
|  | (المتصفح، يوتيوب إلخ)|                         |      (Room Database)    |       |
|  +---------+----------+                         +------------+------------+       |
|            |                                                 ^                    |
|            | (طلب DNS: منفذ 53)                              |                    |
|            v                                                 | (تحديث وتنزيل)     |
|  +---------+----------+      فحص النطاق       +------------+ |                    |
|  |     VpnService     |---------------------->|   محرك     | |                    |
|  |   (TUN Interface)  |<----------------------|  الفلترة   | |                    |
|  +--------------------+   (حظر أو تمرير)      +------------+ |                    |
|                                                              |                    |
+--------------------------------------------------------------v--------------------+
                                                               | (طلب ملفات JSON)
                                                               v
                                                 +--------------------------+
                                                 |    GitHub Pages CDN      |
                                                 |  (ملفات JSON الساكنة)     |
                                                 +--------------------------+
```

### المكونات الأساسية للنظام:
1. **طبقة الشبكة والـ VPN (`core/vpn`):**
   - `SiratVpnService.kt`: تدير دورة حياة الـ VPN المحلي، وتنشئ واجهة الـ TUN الافتراضية.
   - `VpnPacketRouter.kt`: تقرأ الحزم الخام وتمرر حزم DNS (UDP) إلى المحلل.
2. **طبقة تحليل الحزم والـ DNS (`core/dns`):**
   - `DnsPacketParser.kt`: تقوم بتفكيك حزم IPv4/UDP لاستخراج نطاق موقع الويب المطلوب، وتركيب حزمة الرد المحظور (`NXDOMAIN`).
   - `DnsClient.kt`: تمرير الطلبات المسموحة إلى خادم DNS عام آمن (مثل Cloudflare `1.1.1.1`).
3. **طبقة البيانات المحلية والمزامنة (`data/filter`):**
   - `FilterDatabase.kt` & `FilterDao.kt`: قاعدة بيانات Room محلية لتخزين النطاقات النشطة.
   - `SyncManager.kt`: المسؤول عن جلب ملفات JSON الساكنة من خادم GitHub Pages وتفريغ وتحديث الجداول محلياً.
4. **بوابة الملفات الساكنة والاستضافة (بايثون و GitHub Pages):**
   - سكريبت بايثون محلي (`compile.py`) لمعالجة وضغط قوائم الحجب ونشرها على GitHub Pages للعمل كـ CDN ساكن وآمن ومجاني 100%.
5. **طبقة فرض البحث الآمن وتصفية الكلمات (`features/filtering/safesearch`):**
   - إعادة توجيه استعلامات محركات البحث إلى بروتوكول البحث الآمن على مستوى الـ DNS.
6. **واجهة المستخدم (`features/filtering/ui`):**
   - واجهة Jetpack Compose لإدارة إعدادات التصفية وتفعيل الخدمة وعرض السجلات.

## ⚠️ قوانين هامة لمطوري ومحرري الأكواد:
* **لغة البرمجة:** Kotlin بالكامل في جانب الأندرويد.
* **الأداء الخفيف:** يجب ألا تتسبب تصفية طلبات DNS في أي تأخير ملحوظ في سرعة اتصال المستخدم بالإنترنت.
* **الخصوصية أولاً:** لا يتم تسجيل أو إرسال أي نطاقات يزورها المستخدم إلى الخادم، فقط يتم الاستعلام أو تحميل قوائم الحجب من الخادم إلى الهاتف (مزامنة باتجاه واحد).
